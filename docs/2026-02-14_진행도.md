## 1. 프로젝트 개요

### 서비스 설명
**Seed Maker**는 지출 기록 → 절약(씨앗돈) → 배당주 투자 시뮬레이션까지 연결하는 게이미피케이션 기반 예산 관리 앱입니다.

### 핵심 개념
- **씨앗돈**: 예산 대비 절약액 (예산 - 실지출)
- **씨앗 수확**: 1만원당 씨앗 1알
- **배당 시뮬레이션**: 씨앗돈으로 배당주 투자 시 목표 달성 기간 계산
- **게이미피케이션**: 연속 기록 스트릭, 카테고리별 절약 현황

### 타깃 유저
- 30대 FIRE 추구자
- 배당 성장 투자에 관심 있는 직장인
- 기존 가계부 앱 사용에 실패한 경험이 있는 사용자

### 3.3 API Routes

| 엔드포인트 | 메서드 | 기능 |
|-----------|--------|------|
| `/api/dashboard` | GET | 대시보드 집계 데이터 조회 |
| `/auth/callback` | GET | Supabase 인증 콜백 |

#### `/api/dashboard` 응답 구조
```typescript
{
  seedMoney: number;           // 이번 달 씨앗돈
  seedCount: number;           // 씨앗 개수
  budgetTotal: number;         // 총 예산
  expenseTotal: number;        // 총 지출
  byCategory: Array<{          // 카테고리별 상세
    categoryId: string;
    categoryName: string;
    budget: number;
    actual: number;
    saved: number;
  }>;
  currentMonthlyDividend: number;  // 현재 월 배당금
  targetMonthlyDividend: number;   // 목표 월 배당금
  progressPercent: number;         // 목표 진행률
  monthsToGoal: number;            // 목표 달성까지 예상 기간
  streakDays: number;              // 연속 기록 일수
}
```

## 8. 기술 부채 및 해결된 이슈

### 8.1 해결된 이슈 ✅

#### 1. 환경 변수 로딩 문제
**증상**: Server Components에서 `process.env.NEXT_PUBLIC_SUPABASE_*`가 undefined

**시도한 방법**:
- `next.config.ts`의 `env` 블록
- `@next/env.loadEnvConfig`
- `dotenv` 패키지
- 수동 `.env.local` 파싱

**최종 해결**: 
- `next.config.js`의 `env` 블록 제거
- Next.js의 기본 `.env.local` 자동 로딩 사용
- 올바른 Supabase 키 형식 사용 (Publishable Key)

#### 2. BudgetForm 무한 루프
**증상**: `useEffect`에서 "Maximum update depth exceeded" 에러

**원인**: `categories` 객체가 매 렌더링마다 새로 생성되어 의존성 배열이 계속 변경됨

**해결**: 의존성을 `categories?.length`로 변경하여 개수 변경 시에만 실행

#### 3. Cursor IDE 터미널 네트워크 에러
**증상**: `uv_interface_addresses` 시스템 에러

**원인**: Cursor IDE 샌드박스 환경의 제약

**해결**: 외부 터미널(iTerm, Terminal.app)에서 `npm run dev` 실행

### 8.2 남은 기술 부채 ⚠️

#### 1. 타입 정의 개선
**현재 상태**: 일부 컴포넌트에서 `any` 타입 사용

**개선 방안**:
- 모든 `any` 타입을 명시적 타입으로 변경
- Supabase 자동 생성 타입 활용
- 제네릭 타입 적극 활용

**우선순위**: 중간

#### 2. 에러 핸들링 일관성
**현재 상태**: 컴포넌트마다 에러 처리 방식이 다름

**개선 방안**:
