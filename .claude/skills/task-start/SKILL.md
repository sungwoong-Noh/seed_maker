# Task Start Checklist

새로운 작업을 시작하기 전에 다음 항목들을 자동으로 확인합니다.

## 🔍 자동 확인 항목

### 1. 프로젝트 현재 상태
- `docs/CURRENT_STATE.md` 읽기
- 진행 중인 작업 확인
- 다음 예정 작업 파악
- 최근 주요 결정사항 검토

### 2. Git 브랜치 및 상태
- 현재 브랜치 확인 (작업 종류에 맞는 브랜치인지 검증)
- 미커밋 변경사항 있는지 확인
- 최근 커밋 메시지 및 패턴 검토

### 3. 디자인 파일 검토
- `@designs/` 디렉토리의 .pen 파일 확인
- 현재 진행 중인 디자인 작업 파악
- 설계 시스템 규칙 정리

### 4. 작업 계획 수립
- 할 일 목록 작성 (필요시)
- 우선순위 결정
- 예상 범위 설정

## 📝 출력 형식

작업 시작을 위한 다음 정보를 제공합니다:

```
✅ [현재 상태]
- 진행률: XX%
- 다음 작업: ...

✅ [Git 상태]
- 브랜치: <현재 브랜치>
- 최근 커밋: ...

✅ [설계 파일]
- 활성 파일: ...
- 수정 상태: ...

🎯 [오늘의 작업]
- [ ] 작업 1
- [ ] 작업 2
```

## 🚀 사용 방법

```bash
/task-start
/task-start 특정작업명  # 특정 작업에 대해서만 확인
```

## ⚠️ 확인사항

- 브랜치 불일치 시 경고
- 미커밋 변경사항 있으면 주의 요청
- CURRENT_STATE.md 없으면 안내
